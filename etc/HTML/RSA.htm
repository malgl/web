RSA 暗號化 技法

DES System의 단점을 보완한 공개키 System RSA는 상당히 큰 정수의 소인수분해가 힘들다는 점을 이용한 暗號化 System이다.
p와 q가 매우 큰 소수이면 n=p*q의 계산은 용이한 반면, 합성수인 n에서 p와 q를 구하는 것이 아주 어렵다.
System의 개발자인 R. Rivest, A. Shamir, L. Adleman 의 첫 글자를 따서 붙여 RSA, 흔히 공개키 暗號化 System이라 한다.
素數 : 1보다 크며, 1과 그 수 자체 이외의 整數로는 똑 떨어지게 나눌 수 없는 整數(2·3·5·7·11… 따위)

RSA 暗號化 方法

  1. 적당히 비슷하고 큰 소수 p, q를 선택한다.
  2. 공개키 n을 생성한다. (n=pq)
  3. φ를 구한다. φ=(p-1)(q-1)
  4. φ와 서로소인 또 다른 공개키 e를 찾는다. 1<e<φ, gcd(e,φ)=1
  5. 다음 조건을 만족하는 비밀키 d를 만든다. 1<d<φ, ed≡1(mod φ)
    양의 정수 n에 대하여 두 정수 a,b의 차 a-b가 n의 배수 일 때, a와 b는 Modulus n에 합동이라 하고 a≡b(mod n) 으로 나타낸다.
    정수 a를 n으로 나누었을 때의 나머지 r은 a mod n = r로 표현한다. 이때 r을 Modulus n에 대한 a의 잉여라 한다.
  6. {e,n}을 공개키로 공개하고, {d,n}을 개인키로 안전하게 보관한다. p, q, φ 는 공개되지 않도록 한다.
RSA암호화 Encoding (암호화 하려는 수는 m) : E(m) = me mod n = c ( 0 ≤ c < n )
RSA복호화 Decoding : D(c) = cd mod n = m ( 0 ≤ m < n )

예) 암호화와 복호화

  1. 소수 47와 71 선택
  2. 공개키 n=47*71=3337
  3. φ=46*70=3220
  4. 1<e<φ 이고 φ=3220와 서로소인 임의의 정수 공개키 e=79 선정
  5. 1<d<φ, 79*d≡1(mod 3220) 인 비밀키 d=1019
  6. 공개키{79,3337}, 비밀키{1019,3337}
RSA암호화 Encoding (암호화 하려는 수는 688) : E(688) = 68879 mod 3337 = 1570
RSA복호화 Decoding : D(1570) = 15701019 mod 3337 = 688
